Smartsheet filteredEvents API Investigation Summary

Approaches Attempted

1. Power Query (Excel / Power BI)

  • Built a function to call Smartsheet's filteredEvents API per sheet, per day, using since/to date parameters.
  • Iterated through multiple bug fixes including:
    • Type mismatches between Sheet IDs and Event Object IDs.
    • Date/time formatting issues.
    • Handling empty API responses.
  • Extended the solution to batch multiple sheets and multiple date windows into fewer API calls to reduce overall request volume.
  • Implemented cursor-based pagination to retrieve multi-page responses.

2. Python Implementation

  • Rebuilt the solution using Python for greater control and resiliency.
  • Implemented recursive sheet discovery across the workspace.
  • Created batched API calls across sheets and date windows.
  • Added retry logic with exponential backoff.
  • Implemented handling for Smartsheet's documented rate limit (300 requests/minute).
  • Added user-detail enrichment (Name and Email) with caching to avoid redundant lookups.
  • Implemented failed-request retry queues and de-duplication logic to maximize successful data retrieval.

Limitation Encountered

Across every configuration tested—including large batches, small batches, wide date ranges, narrow date ranges, and even the smallest possible request (one sheet with a one-day date window)—the Smartsheet filteredEvents API intermittently returns a 500 Internal Server Error with the message:

"An unexpected error has occurred. Please contact the Support team."

Evidence That the Issue Is Not Client-Side

  • The same sheet succeeds for one day's date window and fails for the very next day's window using an identical request structure.
  • Reducing the request to the smallest possible payload (one sheet and one day) still results in intermittent failures.
  • This rules out payload size, batching strategy, and date-range width as the root cause.
  • Standard mitigation techniques—including smaller batches, retries, exponential backoff, and pagination handling—do not eliminate the issue because the failures originate intermittently on the Smartsheet service.

Conclusion

The investigation indicates that the issue is most likely related to one of the following:
  • A backend issue affecting specific sheets or event data.
  • An account- or workspace-level service instability.
  • An undocumented defect in the Smartsheet filteredEvents endpoint.
Based on the testing performed, no additional changes to the Power Query or Python implementations can reliably prevent these intermittent failures, because the errors occur even for the smallest valid API request and originate from the Smartsheet service rather than the client implementation.
